VÀI CHIÊU TRÒ ĐỂ CODE WEB LOAD NHANH VÀ MƯỢT, TỐC ĐỘ BÀN LÀ

Published in Fact coder
Thg 12 26, 2024
code hay ngủ
Code web thì anh em nào cũng code được, nhưng optimize để cho chạy nhanh, load mượt lại là một bài toán khá phức tạp
Mấy nay, mình có code cái blog nhà trồng bằng AstroJS, xài thử mạng bên này thì cũng khá mượt nên share anh em test thử, sẵn chia sẻ vài tips bên dưới lun
𝐓𝐨̂́𝐢 𝐮̛𝐮 𝐩𝐡𝐢́𝐚 𝐬𝐞𝐫𝐯𝐞𝐫/𝐛𝐚𝐜𝐤-𝐞𝐧𝐝
- Data được truyền từ server tới người dùng với tốc độ ánh sáng, nên càng gần server thì load càng nhanh, càng xa server thì load càng chậm.
=> Chọn server gần người dùng nhất. Ở đây mình dùng server Singapore, nếu có điều kiện thì để ở VN luôn.
- Để build HTML, thay vì mỗi lần load web phải đọc dữ liệu từ database, có thể cache vào memory/redis rồi đọc từ cache.
=> Với blog đơn giản thì... render nguyên 1 cục HTML sẵn luôn, không cần DB hay cache gì hết. (Còn mấy cái cần thay đổi như login/counter thì sao? Mình sẽ chỉ sao hehe)
𝐓𝐨̂́𝐢 𝐮̛𝐮 𝐩𝐡𝐢́𝐚 𝐜𝐥𝐢𝐞𝐧𝐭/𝐟𝐫𝐨𝐧𝐭-𝐞𝐧𝐝
- Load ít file, ít java script, hạn chế số lượng request. Hoặc cần thì inlineCSS vào header luôn để load cho nhanh.
- Dùng CDN (Nói đơn giản là hệ thống server trên toàn thế giới) để lưu file tĩnh như CSS, JS, ảnh,... CDN sẽ lưu copy của các file này vào server gần người dùng nhất.
- Ảnh thì cũng nên optimize lại luôn, nhỏ gọn để load nhanh, lazyload nếu cần để đỡ tốn băng thông
- Prefetch: Khi người dùng load 1 trang web, load sẵn luôn 1 số trang liên quan, để khi người dùng chuyển qua lại giữa các trang thì load nhanh hơn.
=> Ví dụ như hình dưới, khi bạn vào trang chủ, client sẽ load sẵn các blog bên trên luôn .
Do phạm vi bài viết nên mình viết ngắn thui. Hôm sau sẽ có vlog chi tiết hơn kèm code cụ thể nha .
